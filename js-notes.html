JavaScript Interview Questions

  1. Question: What is hoisting in JavaScript?

    Answer: Hoisting is a JavaScript behavior where function and variable declarations move to the top of their respective scope during the compilation phase.

  2. Question: What is Closure in JavaScript?

    Answer: A closure is the combination of a function bundled together with references to its surrounding state. In other words, a closure gives you access to an outer function's scope from an inner function.

  3. Question: What are Promises in JavaScript?

    Answer: Promises in JavaScript are like promises in real life. They represent actions that will finish in the future, either successfully or not. They help manage asynchronous tasks, like fetching data from a server, by providing a way to handle the outcome of these tasks once they're done. Promises have three states: pending (waiting for completion), fulfilled (successfully completed), and rejected (failed to complete). This makes it easier to write asynchronous code that is easier to read and understand.

  4. Question: What are Callback Functions? What is their use?

    Answer: Callback functions are functions passed as arguments to other functions, and they are executed after the completion of a specific task. They are commonly used in asynchronous operations, event handling, and more.

  5. Question: What is the purpose of the “async” and “await” keywords in JavaScript?

    Answer: The “async” keyword is used to declare an asynchronous function, which allows the use of the “await” keyword inside it. “await” is used to pause the execution of an async function until a Promise is resolved, simplifying asynchronous code and avoiding callback hell.

  6. Question: What are Event Loop?

    Answer: Event Loop: The event loop is a core mechanism in JavaScript for handling asynchronous operations. It continuously checks the call stack and task queue, moving tasks from the queue to the stack when the stack is empty. This allows JavaScript to handle non-blocking I/O operations efficiently.

  7. Question: What are this Keyword?

    Answer: The “this” keyword refers to the object that owns the current code being executed. Its value is determined by how a function is called or how an object is constructed.

  8. Question: What is the difference between map() and forEach() array methods of an array?

    Answer: The map() method is used when you want to modify each element of an array and create a new array with the modified values, forEach() method is used when you want to perform some operation on each element of an array without creating a new array.

  9. Question: What is Modules?

    Answer: Modules - Modules allow you to organize and encapsulate JavaScript code into reusable modules that can export and import functionality.

  10. Question: What is JSON, and what does it consist of?

    Answer: JSON is a lightweight data interchange format. It consists of key-value pairs.

  11. Question: What is the purpose of the “localStorage” and “sessionStorage” objects?

    Answer: The “localStorage” and “sessionStorage” objects provide web developers with a way to store key-value pairs locally on a user’s browser. The “localStorage” data persists across sessions, while the “sessionStorage” data is cleared when the session ends.

  12. Question: How do you handle asynchronous operations in JavaScript?

    Answer: Asynchronous operations in JavaScript are typically handled using callbacks, promises, or the newer async/await syntax. Callbacks were the traditional approach, while promises and async/await provide more structured and readable code.

  13. Question: What are the differences between JavaScript’s “null” and “undefined”?

    Answer: “null” represents the deliberate absence of an object value, while “undefined” indicates an uninitialized variable or a variable that lacks a value. “null” is an assigned value, while “undefined” is a type in JavaScript.

  14. Question: What is the purpose of the “apply” and “call” methods in JavaScript?

    Answer: Both “apply” and “call” are used to invoke a function with a specified context (the value of “this”). The “apply” method accepts arguments as an array, while the “call” method accepts arguments individually.

  15. Question: What is short-circuit evaluation in JavaScript?

    Answer: Short-circuit evaluation stops the execution as soon as the result can be determined without evaluating the remaining sub-expressions.

  16. Question: What determines the type of a variable?

    Answer: The data type determines the type of a variable.

  17. Question: What is type coercion in JavaScript?

    Answer: Type coercion is the automatic conversion of values between data types during operations or comparisons, such as string and number concatenation and the use of comparison operators.

  18. Question: How is the spread operator (...) used in JavaScript?

    Answer: The spread operator is used to expand or spread elements from an iterable (such as an array, string, or object) into individual elements. It is used for copying an array, merging arrays, and passing multiple arguments to a function.

  19. Question: What is the rest operator used for in JavaScript?

    Answer: The rest operator is used in function parameters to collect all remaining arguments into an array.

  20. Question: What is the difference between named and anonymous functions? When to use what in applications?

    Answer: Difference between Named and Anonymous Functions:
    Named Functions: Have a name, can be called before the function declaration.
    Anonymous Functions: Often used as expressions, defined at runtime, and can't be called before the declaration.

  21. Question: What is a function expression in JS?

    Answer: Function Expression in JavaScript:
    A function expression is a way to define a function within an expression rather than as a declaration. It can be named or anonymous.

  22. Question: What are Arrow Functions in JS? What is their use?

    Answer: Arrow functions are a concise way to write anonymous functions. They have a shorter syntax and do not bind their own this value.

  23. Question: What is a Higher-order function In JS?

    Answer: A higher-order function is a function that takes another function as an argument or returns a function as a result. They enable functional programming concepts like mapping, filtering, and reducing.

  24. Question: What is the difference between arguments and parameters?
